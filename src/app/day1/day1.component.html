Data Binding

Data binding allows communication between a component and the DOM, making it very easy to define interactive applications without worrying about pushing and pulling data. There are four forms of data binding and they differ in the way the data is flowing -Interpollation, Property Binding, Event Binding, 2-Way Binding



Interpollation: {{value}}


<h5>Today's Date: {{today}}</h5>
Today's Date: {{today}}


Property Binding: [property]=”value”


Binds data to a property of an element. The trick is to put the property onto the element wrapped in square brackets: [property]

helloImg: String = "/assets/Hello!!!.png" 
          |
          |
<img [src]='helloImg'>


Event binding: (event)=”function”

When a specific DOM event happens (eg.: click, change, keyup), call the specified specified method in the component. 

{{eventBindingCode}}
         |
         |
<button (click)="sayHello()">Click me</button>


2-Way Binding: [(ngModel)]=”value”

Using what’s called the banana in a box syntax, two-way data binding allows to have the data flow both ways.

Import FormsModule in app.module

myText: String='Enter Text';
     |
     |
<input type="text" [(ngModel)]="myText"> {{myText}}
